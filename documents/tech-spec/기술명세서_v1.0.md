# 기술명세서 v1.0

**프로젝트**: 남구이야기 (Namgu Story)  
**버전**: 1.0  
**작성일**: 2024-12-22  
**기술 기준**: Flutter + Firebase

---

## 1. 아키텍처
- **Frontend**: Flutter (Dart)
- **상태관리**: Riverpod
- **네비게이션**: GoRouter
- **Backend**: Firebase (Auth, Firestore, Functions, Storage, FCM)
- **지도**: Kakao Map 또는 Naver Map API

## 2. 인증
- **이메일/비밀번호**: Firebase Auth
- **카카오 로그인**: Custom Token 방식
  - 문서: `documents/tech-spec/firebase-kakao-auth.md`

## 3. 데이터
- **Firestore 스키마**: `documents/data-model/firestore-schema.md`
- **권한**: Security Rules 기반, 민감 로직은 Functions에서 처리

## 4. 클라이언트 구조
- `lib/theme`: 디자인 시스템
- `lib/widgets`: 공통 컴포넌트
- `lib/screens`: 화면 단위 UI
- `lib/features`: 도메인 로직(쿠폰/미션/친구/설정 등)

## 5. 배포
- Android: `flutter build appbundle --release`
- iOS: `flutter build ios --release`

---

추가 상세 구현은 각 기능별 문서를 우선 기준으로 삼는다.
